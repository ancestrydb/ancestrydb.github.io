Why ancestryDB?

Built for genealogy, it handles large, branching family trees, uncertain dates and sources, and keeps provenance for every change.

  • Provenance-aware storage
  • Temporal queries (approximate dates)
  • Flexible schema for events, people, documents
  • Compact binary on-disk format

Drivers & Integrations

Official driver for Python focuses on stability and clear semantics.

from ancestrydb import Ancestry

# Connect to server
conn = Ancestry.connect(host='localhost', port=3863)

# Create person with sources
person = conn.create_person(name='John Doe', sources=['census_1890'])

# Add relationship
conn.add_relationship(person, 'Jane Doe', relation='spouse')

# Query people born around 1850
results = conn.search(birth_year={'approx': (1850, 5)})
for r in results:
    print(r.name, r.birth_year)

Advanced topics

  • Replication topology: multi-master with vector-clock merges for eventual consistency or quorum writes for strong consistency.
  • Provenance and sources: attach structured citations and keep an immutable history for each node.
  • Storage: compact binary tree format with on-disk sharding per top-level surname for large datasets.
  • Migrations: safe schema migrations and tooling to import GEDCOM and reconcile duplicate records.
